**Critical Finding:** Configuration overhead consuming 20-40% of development time—precisely the activation friction an ADHD-focused system should eliminate.

## The Core Contradiction

Global CLAUDE.md warns "Sometimes over-engineers solutions - keep it simple" while configuration system itself violates this through:
- Active-project feature solving non-existent problem
- Multiple CLAUDE.md files requiring constant synchronization

## Why Projects Expand

Your thinking pattern (connecting concepts, seeing implications, bouncing between domains) isn't a bug—**it's the same capability you're trying to enable for users.**

You're doing Flow Mode thinking *while building the tool for Flow Mode thinking*. Every connection suggests another feature.

**The problem:** No way to defer these connections. Everything gets added to current project instead of being captured for later.
