## Core Assumption: ADHD Brains Are Not Broken

> "ADHD is not a problem of knowing what to do; it is a problem of doing what you know." â€” Russell Barkley

IES is built on a foundational insight: **ADHD brains are interest-based, emotionally-driven, and connection-making machines.** The system must work WITH these traits, not against them.

Traditional knowledge tools assume:
- Users can start from a blank page
- Importance determines motivation
- Organization precedes retrieval
- Linear thinking is the default
- Memory works like a file cabinet

**IES assumes the opposite:** ADHD cognition requires multiple entry points, resonance-based capture, visible progress, non-judgmental status lifecycle, and external working memory.

## Six Core ADHD Cognitive Assumptions

### 1. Cannot Start from a Blank Page

**Research Source:** Russell Barkley - Executive Functions
**Key Insight:** "The policeman is asleep" â€” prefrontal cortex can't prioritize/inhibit without external structure

**ADHD Challenge:** Task initiation requires external triggers. A blank page paralyzes.

**How IES Addresses This:**
- **Dashboard Suggestions:** "Most Connected" and "Novel Concepts" provide starting points
- **Favorite Problems:** Feynman-style anchor questions (e.g., "How do people change?") create stable entry points
- **Recent Journeys:** "Continue where you left off" with breadcrumb trails
- **Mood-Based Access:** "Show me something exciting" or "Show me something calming" based on energy level
- **Serendipitous Discovery:** Random meaningful entry points via graph traversal

**Implementation:**
- `.worktrees/siyuan/ies/plugin/src/views/Dashboard.svelte` â€” Suggestion cards with clickable concepts
- `ies/backend/src/ies_backend/api/personal.py` â€” `/personal/sparks/unvisited` endpoint surfaces forgotten captures

### 2. Interest-Based Nervous System (Not Importance-Based)

**Research Source:** Tamara Rosier - Your Brain's Not Broken
**Key Insight:** ADHD motivation follows emotional resonance, not objective value. Shame is a "malicious motivator."

**ADHD Challenge:** "This is important" doesn't generate energy. "This sparks something" does.

**How IES Addresses This:**
- **Resonance Signals:** 8 emotional tags (`curious`, `excited`, `surprised`, `moved`, `disturbed`, `unclear`, `connected`, `validated`) capture WHY something matters
- **Energy Levels:** Navigate by current capacity (low/medium/high), not forced engagement
- **Non-Judgmental Status:** Growth metaphor (`captured â†’ exploring â†’ anchored`) avoids shame triggers like "incomplete" or "draft"
- **Quick Capture:** `spark` entity type enables save-now-classify-later (zero friction)

### 3. Nonlinear/Associative Thinking Patterns

**Research Source:** Cognitive architecture research (spreading activation models)
**Key Insight:** ADHD minds make unexpected connections. Traditional hierarchical organization fights this strength.

**ADHD Challenge:** Forcing linear organization kills creativity. "Where does this go?" becomes a blocker.

**How IES Addresses This:**
- **Graph-Based Navigation:** Follow threads wherever they lead, not forced taxonomy
- **Multiple Relationship Types:** `resonates_with`, `sparked_by`, `led_to_discovery`, `addresses_problem` preserve associative connections
- **Thread Entities:** Capture exploration journeys across sessions without forcing premature categorization
- **Flow Mode:** Browse knowledge graph visually, clicking relationships to explore

### 5. Insight Through Guided Questioning (Not Information Retrieval)

**Research Source:** Socratic method + cognitive behavioral therapy literature
**Key Insight:** ADHD minds excel at synthesis when given scaffolding. Generic prompts don't help.

**ADHD Challenge:** "Figure it out yourself" overwhelms. "What are the main categories here?" scaffolds.

**How IES Addresses This:**
- **9 Question Classes:** Schema-Probe, Boundary, Dimensional, Causal, Counterfactual, Anchor, Perspective-Shift, Meta-Cognitive, Reflective-Synthesis
- **Cognitive Hints:** Class-specific guidance (e.g., Schema-Probe: "Try listing the main categories, components, or buckets")
- **Response Starters:** Optional sentence starters reduce friction (e.g., "The main parts are...")
- **Mode-Specific Questions:** Learning mode uses Schema-Probe + Causal; Ideating uses Counterfactual + Perspective-Shift

**Implementation:**
- `.worktrees/siyuan/ies/plugin/src/views/ForgeMode.svelte` â€” `QUESTION_CLASS_HINTS` constant (lines 174-220) with hint text, prompts, response starters
- `ies/backend/src/ies_backend/api/question_engine.py` â€” Question generation pipeline with class tagging
- `ies/backend/src/ies_backend/schemas/question_engine.py` â€” `QuestionClass` enum, `APPROACH_TO_CLASSES` mapping

### 6. Understanding Through Movement (Exploration, Not Reading)

**Research Source:** Embodied cognition literature
**Key Insight:** ADHD minds understand by doing, not by passive consumption

**ADHD Challenge:** Reading linearly feels like trudging. Browsing, clicking, discovering = flow state.

## The 6-Dimension User Cognition Model

**Note:** Designed but not yet fully implemented in profile tracking.

IES personalizes questions and suggestions using a 6-dimension cognitive profile (0-1 scale):

| Dimension | Low End | High End | Purpose |
|-----------|---------|----------|---------|
| **structure_preference** | Open exploration | Scaffolded guidance | Determines template structure |
| **pace** | Slow/deep | Fast/broad | Adjusts question frequency |
| **ambiguity_tolerance** | Needs closure | Sits with uncertainty | Affects when to synthesize |
| **abstraction_level** | Concrete examples | Abstract concepts | Shapes question abstraction |
| **verification_need** | Intuitive trust | Evidence-based | Determines anchor question ratio |
| **novelty_preference** | Familiar patterns | Surprising connections | Affects suggestion diversity |

**Usage:**
- **Template Defaults:** High structure preference â†’ more guided sections
- **Question Class Selection:** Low abstraction â†’ more Anchor questions (concrete examples)
- **Mode Transition Triggers:** Low pace â†’ longer time before suggesting mode switch

**Implementation Status:**
- Schema exists in backend
- Profile creation/retrieval works
- **Not yet validated:** Does profile actually update from conversations? Does it affect question generation?

**Future Work:**
- Automatic profile updates from session patterns
- A/B testing to validate profile impact on question quality

## Design Principles (Derived from Assumptions)

1. **Resonance Over Importance** â€” Capture emotional engagement, not just taxonomic accuracy
2. **Multiple Entry Points** â€” Mood, question, energy, resonance, recency, serendipity
3. **Visible Progress** â€” Breadcrumb trails as treasure maps, not audit logs
4. **Low Friction, High Return** â€” Capture effortless, organization automatic, retrieval interest-based
5. **Context Preservation** â€” Save "why now?" alongside "what"
6. **Non-Judgmental Lifecycle** â€” Growth metaphor over productivity metrics
7. **Interest-Following Navigation** â€” Graph traversal feels like following curiosity

## What This Means for Future Development

**Any feature addition must answer:**
1. Does this work WITH ADHD cognition or against it?
2. Does this reduce cognitive load at decision points?
3. Does this provide multiple valid entry points?
4. Does this surface interesting connections or require manual organization?
5. Does this preserve context across interruptions?
6. Does this support non-linear navigation?

**Anti-Patterns to Avoid:**
- Forced categorization before capture
- Single "correct" way to find things
- Productivity metrics that trigger shame
- Linear-only navigation
- Blank-page starting points
